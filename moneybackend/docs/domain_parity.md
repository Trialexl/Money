# Паритет доменной модели с 1С

Этот документ фиксирует принятые решения по областям, где 1С и Django не совпадают 1:1, но расхождение должно быть понятным и контролируемым.

## Контрагенты

Целевое решение:

- отдельный справочник `Контрагенты` в Django пока не вводится
- роль контрагента выполняет `users.CustomUser`
- `CustomUser` считается целевой моделью для:
  - `НаименованиеПолное -> full_name`
  - `Юрлицо -> status`
  - `ИНН -> tax_id`

Что сознательно не переносится как отдельная сущность:

- реквизит `Пароль` из 1С не хранится как поле контрагента
- вместо него используется штатный пароль пользователя Django

Когда понадобится отдельный справочник:

- если в документах появятся ссылки на контрагентов, не являющихся учетными записями
- если потребуется хранить несколько организаций/контрагентов на одного пользователя
- если `users.CustomUser` начнет смешивать аутентификацию и справочные данные продукта

## Графики и шапка документа

Принятое правило:

- поля шапки документа (`amount`, `date_start`, `amount_month`) остаются источником данных для создания и регенерации графика
- строки графика являются источником данных для регистров там, где это уже реализовано в коде
- прямое редактирование строк графика разрешено и немедленно пересобирает регистры
- прямое редактирование строк графика не обязано автоматически пересчитывать поля шапки

Практический смысл:

- `generate-graphics` используется как штатный способ синхронно обновить и шапку, и строки
- ручное редактирование строк рассматривается как точечная корректировка графика, а не как пересборка документа целиком
- это правило отдается в API в поле `graphic_contract` и показывается в admin на карточках документов с графиком

Детализация по типам документов:

- `Expenditure`: шапка хранит сумму фактического расхода, график управляет только бюджетным распределением
- `Transfer`: шапка хранит сумму фактического перевода, график управляет только бюджетным распределением, остаток остается на дате документа
- `Budget`: шапка является шаблоном генерации графика, а строки графика становятся источником данных для регистров
- `AutoPayment`: шапка является шаблоном генерации графика, а строки графика становятся источником данных для движений и бюджетных регистров

## Что считать эталоном 1С

Эталонными считаются:

- движения по регистрам
- расчеты dashboard и отчетов
- обязательность и дефолты, если они влияют на бизнес-результат

Не считаются обязательным эталоном:

- механика проведения как платформенная особенность 1С
- точное совпадение UI форм
- наличие в Django отдельных сущностей только потому, что они были отдельными объектами метаданных в 1С

## Регрессионные сценарии

Для защиты от дрейфа относительно 1С в автотестах зафиксированы сценарии:

- приход
- расход без распределения
- расход с распределением
- перевод без бюджета
- перевод с бюджетным распределением
- бюджет расходов на несколько месяцев
- бюджет доходов на несколько месяцев
- автоплатеж-расход
- автоплатеж-перевод
